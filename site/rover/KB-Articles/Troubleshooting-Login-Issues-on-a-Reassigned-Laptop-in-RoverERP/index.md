# Troubleshooting Login Issues on a Reassigned Laptop in RoverERP

<PageHeader />

---

## Resolution Steps

### 1. Clear Cached Credentials and User Profiles

- Log out of any existing RoverERP sessions on the laptop
- Remove or reset any cached credentials or user profiles associated with the previous user
- Restart the laptop

### 2. Verify Network Connectivity

- Ensure the laptop has a stable network connection and can reach the RoverERP server
- Test connectivity by accessing other network resources or websites

### 3. Check Device Registration and Permissions

- Confirm that the laptop is properly registered and authorized for use with RoverERP
- If device registration is required, update or re-register the laptop under the new user's credentials

### 4. Reinstall RoverERP with Administrative Rights

- Uninstall RoverERP completely
- Reinstall RoverERP using administrative privileges to ensure all components are properly installed
- After installation, attempt to log in with the new user's credentials

### 5. Review Error Screens and Logs

- Take note of any error messages or codes displayed during the login attempt
- Review RoverERP logs (if accessible) for additional details on the failure

### 6. Contact Support if the Issue Persists

- If the problem continues after completing the above steps, contact RoverERP support with details of the issue and any error messages encountered
